Project brief

What is Econ Trends?

A reproducible research workflow for tracking whether top economics journal articles are framed around causal questions, predictive questions, or neither.

Scope

Five general-interest journals

American Economic Review, Quarterly Journal of Economics, Journal of Political Economy, Econometrica, and Review of Economic Studies.

Period

1975-2025 article panel

The current working dataset covers publication years 1975 through 2025, with a closer first-batch inspection of 2023-2025.

Question

How research framing changes

The dashboard reports descriptive shifts in the share of articles classified as causal, predictive, other, or insufficient text.

Causal Estimating, identifying, or interpreting causal effects.
Predictive Prediction, forecasting, classification, performance, or out-of-sample accuracy.
Other Papers that do not clearly fit the causal or predictive definitions.
Insufficient text Records where available title/abstract/full-text evidence is not enough to classify reliably.
Data collection

How we collect the article data

The project starts from public bibliographic metadata, then constructs a deduplicated article-level panel for the top-five journal scope before any classification is run.

Source collection

Crossref / OpenAlex
01

Define the five journals with ISSN-L, print/electronic ISSNs, publisher names, and OpenAlex source IDs in config/journals.yml.

02

Query Crossref by journal ISSN and publication year to collect DOI-centered bibliographic records for 1975-2025.

03

Query OpenAlex Works by verified source IDs and years to recover article metadata, abstracts where available, authorships, concepts, topics, keywords, and DOI links.

Article panel construction

deduplication
04

Standardize Crossref and OpenAlex records into a common source-record table with source provenance columns.

05

Deduplicate to one article record by normalized DOI when available, otherwise by normalized title, journal, and publication year.

06

Track missingness and provenance for key fields such as title, abstract, DOI, author names, journal, volume, issue, pages, keywords, and source-provided article type.

Coverage boundary The current dataset is a public-source metadata panel, not a final archival journal database. Crossref and OpenAlex records update over time, JEL coverage is currently missing, and abstract availability varies by journal and year.

Replication

Sharing and replication

The public repository contains the code, configuration, prompts, tests, documentation, and current dashboard snapshot. Large raw and intermediate data artifacts are kept out of git and should be distributed separately for full computational replication.

Run path

Rebuild from archived inputs

Restore the archived data/raw, data/intermediate, and data/final folders, install dependencies, then run the documented Makefile targets for tests, validation diagnostics, and trend tables.

Data boundary

Public code, separate data package

The repository does not commit raw API caches, PDF/OCR caches, reviewer files, or generated output tables. That boundary keeps the GitHub project readable while preserving a path for a formal replication archive.
