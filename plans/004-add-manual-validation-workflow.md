## Why this matters

Classification cannot be credible without a manual validation sample. The sample must be stratified so it includes obvious causal papers, obvious predictive papers, likely other papers, ambiguous papers, short-text cases, and records from each journal/year. This plan creates the validation artifacts but does not require the user to complete manual labeling during implementation.

## Steps

### Step 1: Implement stratification helpers

Create `code/04_classify/validation_sample.py`.

Required functions:

- `assign_validation_strata(df)` - adds strata columns.
- `sample_validation_rows(df, sample_size, seed)` - returns deterministic sample.
- `make_label_template(df)` - adds empty manual-label columns.

Strata to include:

- `journal_short`
- `publication_year`
- current predicted category
- current confidence
- `has_abstract`
- score bins for causal and predictive indicators if present
- ambiguous indicator when causal and predictive indicators are both positive or confidence is low

Manual label columns:

- `validation_id`
- `manual_label` with empty values
- `manual_confidence` with empty values
- `manual_notes` with empty values
- `reviewer_id` with empty values
- `review_date` with empty values

### Step 2: Add deterministic sampling runner

Create `run_validation_sample.py`.

Arguments:

- `--input`, required
- `--output`, default `data/intermediate/manual_validation_sample.csv`
- `--sample-size`, default `300`
- `--seed`, default `20260628`

Behavior:

- Read classified CSV.
- Determine available category/confidence columns. Prefer `llm_category` and `llm_confidence` if present and `llm_status == ok`; otherwise use `causal_predictive_category` and `classification_confidence`.
- Stratify and sample deterministically.
- Always include all rows with `causal_predictive_category == insufficient_text` up to a capped share, or sample them if there are too many.
- Write a CSV designed for human review.

**Verify**: run the sample command above. If full LLM output does not exist, use the fallback input from rule-based output and document that in final notes.

### Step 3: Add validation tests

Create `tests/test_validation_sample.py`.

Test cases:

- Same seed returns same `article_id` order.
- Different seed can return a different sample.
- Sample includes rows from multiple journals when available.
- Low-confidence/ambiguous rows are included when available.
- Required manual label columns exist and are empty.

**Verify**: `python3 -m unittest discover -s tests` -> exits 0.

### Step 4: Write manual validation codebook

Create `docs/manual_validation_codebook.md`.

Include:

- Category definitions.
- How to label `causal`, `predictive`, `other`, and `insufficient_text`.
- Instruction to use title and abstract only for main validation.
- Examples of borderline cases:
  - empirical estimate of treatment effect -> causal
  - forecasting/predictive performance/out-of-sample comparison -> predictive
  - pure theory, descriptive measurement, institutional narrative -> other unless primary question is causal or predictive
  - missing/too short abstract -> insufficient_text unless title is clearly decisive and project owner approves title-only labels
- Fields reviewers must fill.
- Rule that disagreements should be adjudicated rather than averaged.

**Verify**: `sed -n '1,260p' docs/manual_validation_codebook.md` -> includes all four category definitions.

## Maintenance notes

The validation sample should be regenerated when the prompt or rule dictionary changes. Reviewers should check that ambiguous and low-confidence rows are intentionally oversampled; a purely random sample will understate the classification problem.

Post-implementation update on 2026-06-29: reviewer HTML forms now show the primary-focus label rubric inline, and `validate_manual_label_values` rejects completed labels that lack `manual_confidence`, `reviewer_id`, or ISO `review_date`. Calibration submissions also validate `calibration_id`, `validation_id`, and `article_id` against the active calibration packet and reject duplicate completed rows from the same reviewer. This keeps main validation, calibration, and overlap submissions auditable before labels can move the validation gate.
